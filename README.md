# Resume Marker - AI简历生成器

## 📖 使用指南

### 1. 快速开始

#### 安装依赖
```bash
npm install
```

#### 启动开发服务器
```bash
npm run dev
```

访问 `http://localhost:3000/` 即可使用。

---

### 2. 配置 API Key

本工具使用 **MiniMax API** 生成简历，需要配置 API Key：

1. 在右侧「AI 助手」面板点击「请先配置 API Key」
2. 输入你的 MiniMax API Key
3. 选择模型（目前支持 MiniMax-M3）
4. 点击「保存配置」

> **获取 API Key**：访问 [MiniMax 开放平台](https://api.minimaxi.com/)

---

### 3. 生成简历

#### 方式一：使用预设模板

1. 在左侧「提示词模板」区域，选择岗位分类：
   - 技术类（前端、后端、全栈）
   - 产品类（产品经理）
   - 设计类（UI/UX设计师）
   - 市场类（市场营销）
   - 应届生（实习生）

2. 选择具体模板（如「前端工程师」）

3. **编辑提示词（可编辑部分）**
   
   > 💡 **提示**：提示词分为两部分：
   > - **固定部分**（隐藏）：系统自动添加的API调用指令，如"输出完整HTML"、"A4尺寸"等
   > - **可编辑部分**（可见）：您可以修改简历的设计风格、模块要求、配色等内容

4. 点击「生成简历」按钮

5. AI 会自动生成完整的 HTML 简历，并在中间区域预览

#### 方式二：自定义提示词

1. 选择「自定义提示词」模板

2. 在文本框中输入你的需求，例如：
   ```
   生成一份前端工程师的简历，要求：
   - 设计风格：现代简约
   - 主色调：深蓝色
   - 突出 React 和 TypeScript 经验
   - 包含 GitHub 和作品集链接
   ```

3. 点击「生成简历」

---

### 4. 优化简历

生成简历后，你可以通过对话方式让 AI 优化简历：

1. 在右侧「AI 助手」面板输入修改需求，例如：
   - "把工作经历改得更专业一些"
   - "添加一个项目经验 section"
   - "把技能列表用表格展示"
   - "整体色调改成绿色"

2. 点击「发送」或按 `Ctrl + Enter`

3. AI 会重新生成 HTML 并更新预览

> **提示**：AI 会展示思考过程，点击「🤔 思考过程」可以查看。

---

### 5. 导出简历

#### 导出 PDF
点击顶部「导出 PDF」按钮，自动生成并下载 PDF 文件。

#### 打印
点击「打印」按钮，使用浏览器打印功能。

#### 复制 HTML
点击「复制 HTML」按钮，将生成的 HTML 代码复制到剪贴板。

### 6. 提示词编写技巧

#### 好的提示词包含：

1. **设计风格**：现代简约、专业商务、创意时尚等
2. **颜色主题**：主色调、辅助色
3. **必备模块**：个人信息、工作经历、项目经验、技能等
4. **布局要求**：单栏/双栏、字体大小等
5. **特殊要求**：技能展示方式（标签/进度条/图表）

> **注意**：以下内容是系统自动添加的，无需在提示词中重复：
> - "请直接输出完整的HTML代码"
> - "A4纸张大小"
> - "内联CSS样式"
> - "使用语义化HTML5标签"

#### 示例提示词：

```
设计风格：专业稳重，结构清晰
主色调：深绿色 (#2f855a) + 白色
字体：等宽字体风格，14-16px

必备模块：
1. 个人信息
2. 技术技能：编程语言、数据库、中间件、云服务
3. 工作经历：突出系统架构设计、性能优化
4. 项目经验：包含系统架构描述、QPS优化数据
5. 教育背景

布局建议：表格或flex布局展示技能熟练度
```

### 7. 常见问题

#### Q: 生成的简历样式错乱？
A: 系统已自动添加A4尺寸和完整HTML输出要求，确保提示词中设计风格描述清晰。

#### Q: PDF 导出后分页截断？
A: AI 生成的 HTML 已包含 `page-break-inside: avoid` 样式，如仍有问题，可在提示词中强调布局要求。

#### Q: 如何保存当前进度？
A: 生成的 HTML 会自动保存到浏览器本地存储，刷新页面后会自动恢复。

#### Q: API 调用失败？
A: 检查以下几点：
- API Key 是否正确
- 网络是否畅通
- MiniMax 账户余额是否充足

#### Q: 提示词中的固定部分可以修改吗？
A: 固定部分（如输出格式要求）是系统自动添加的，用户无法修改，这确保了生成的简历符合标准格式。

### 8. 技术架构

#### 核心技术栈
- **前端框架**：Vue 3 + TypeScript
- **UI 组件库**：Element Plus
- **状态管理**：Pinia
- **AI API**：MiniMax API (OpenAI 兼容)
- **PDF 生成**：html2pdf.js

#### 提示词模板系统
```
提示词模板结构：
├── fixedPrompt    # 固定提示词（隐藏）- API调用指令、输出格式要求
└── editablePrompt # 可编辑提示词（可见）- 设计风格、模块要求、配色等
```

#### 工作流程
```
选择模板 → 编辑可编辑提示词 → 组合 fixedPrompt + editablePrompt → AI 生成 HTML → 实时预览 → 对话优化 → 导出 PDF
```

#### 目录结构
```
src/
├── components/          # Vue 组件
│   ├── PromptTemplateSelector.vue  # 提示词选择器
│   ├── ResumePreview.vue           # 简历预览
│   └── ChatPanel.vue               # AI 对话面板
├── config/              # 配置文件
│   └── promptTemplates.ts          # 提示词模板（含fixedPrompt和editablePrompt）
├── stores/              # 状态管理
│   ├── resume.ts                   # 简历数据
│   └── chat.ts                     # 对话状态
├── views/               # 页面视图
│   └── EditorView.vue              # 主编辑页面
├── types/               # TypeScript 类型定义
└── utils/               # 工具函数
    └── pdf.ts                      # PDF 导出
```

### 9. 开发说明

#### 本地开发
```bash
npm run dev        # 启动开发服务器
npm run build      # 构建生产版本
npm run preview    # 预览生产版本
```

#### 自定义提示词模板

编辑 `src/config/promptTemplates.ts`，添加新模板：

```typescript
export const promptTemplates: PromptTemplate[] = [
  {
    id: 'my-template',
    name: '我的模板',
    description: '模板描述',
    category: 'tech',
    icon: '💻',
    prompt: '',              // 完整提示词（自动组合）
    fixedPrompt: COMMON_FIXED_PROMPT,  // 固定部分（可选，默认使用通用固定提示词）
    editablePrompt: `        // 可编辑部分
      设计风格：...
      主色调：...
      必备模块：...
    `
  }
]
```

**提示词模板字段说明**：
- `fixedPrompt`：固定提示词，包含API调用指令和输出格式要求，用户不可见
- `editablePrompt`：可编辑提示词，包含设计要求、模块配置等，用户可修改
